Introduction

Terminal Access for NVDA is an add-on that provides enhanced terminal accessibility for Windows users. It brings Terminal Data Structure Reader (TDSR) functionality directly into NVDA, enabling screen reader users to efficiently navigate and interact with command-line interfaces.

This add-on integrates seamlessly with NVDA's built-in speech synthesis and provides a comprehensive set of navigation and reading commands specifically designed for terminal usage.

Features

Core Navigation Features

Line Navigation

Move through terminal output line by line to review command results at your own pace.

Example Usage: After running dir or ls, use NVDA+Alt+I to hear the current line, NVDA+Alt+O to move to the next line, and NVDA+Alt+U to go back to the previous line. This is perfect for reviewing file listings, log output, or any multi-line command results.

Word Navigation

Navigate by word within the terminal for faster review of long command lines and paths.

Example Usage: When reviewing a long path like C:\Users\Username\Documents\Projects\MyApp\src\components\Button.jsx, use NVDA+Alt+K to hear the current word (e.g., "components"), NVDA+Alt+L to move to the next word (e.g., "Button"), and NVDA+Alt+J to go back. This is much faster than character-by-character review for long lines.

Character Navigation

Move character by character with phonetic spelling support for precise review.

Example Usage: When verifying a complex filename like data_2024-01-15_v2.json, use NVDA+Alt+Comma to hear each character. Press NVDA+Alt+Comma twice to hear phonetic spelling (e.g., "alpha" for "a", "bravo" for "b"), which is helpful for distinguishing similar-sounding letters. Press three times to hear the character code (e.g., "Character 97, hex 61, a").

Advanced Reading Features

Continuous Reading (Say All)

Read continuously from the current position to the end of the terminal buffer.

Example Usage: After running a command that generates lots of output like git log or npm install, position your review cursor at the beginning with NVDA+Alt+PageUp, then press NVDA+Alt+A to hear all the output read continuously. Press any key to stop reading at any time.

Indentation Detection

Announce the indentation level of the current line, essential for Python, YAML, and other indentation-sensitive formats.

Example Usage: When reviewing Python code or YAML configuration files in the terminal, press NVDA+Alt+I twice on any line to hear its indentation. For example, "4 spaces" or "2 tabs" helps you understand the code structure without seeing it visually.

Position Announcement

Reports your current row and column position in the terminal buffer.

Example Usage: When navigating through a large file or log output, press NVDA+Alt+P to hear something like "Row 142, column 25". This helps you keep track of where you are in long outputs or when sharing position information with others.

Real-Time Feedback Features

Cursor Tracking

Automatically announces the character at the cursor position when it moves.

Example Usage: When using arrow keys to navigate through a command you're editing, cursor tracking announces each character as you move, helping you verify your position. For example, if you're navigating through git commit -m "message", moving right through it announces "g", "i", "t", "space", "c", and so on.

Key Echo

Hear characters as you type them for immediate feedback.

Example Usage: As you type npm install express, each keystroke is announced: "n", "p", "m", "space", "i", "n", "s", "t", "a", "l", "l", and so on. This helps catch typos immediately and confirms you're typing correctly.

Symbol Processing

Speak symbol names instead of the symbols themselves for clarity.

Example Usage: When typing a shell script like #!/bin/bash, with symbol processing enabled, you hear "hash exclamation slash bin slash bash" instead of just the symbol sounds. This is especially useful for commands with many special characters like grep -r "pattern" . | sort -u > results.txt.

Advanced Features

Quiet Mode

Temporarily disable automatic announcements when dealing with verbose output.

Example Usage: Before running a command that produces lots of output like compiling code or installing packages, press NVDA+Alt+Q to enable quiet mode. The command runs without NVDA announcing every cursor movement or output change. When done, press NVDA+Alt+Q again to re-enable announcements and review the output manually.

Selection and Copy

Select and copy terminal content using mark-based selection or legacy copy mode.

Example Usage: To copy a specific section of output:
  1. Navigate to the start of the text you want to copy
  2. Press NVDA+Alt+R to set the start mark (you'll hear "Mark start set")
  3. Navigate to the end of the text
  4. Press NVDA+Alt+R again to set the end mark (you'll hear "Mark end set")
  5. Press NVDA+Alt+C to copy the selection as continuous text, or NVDA+Alt+Shift+C for rectangular (column-based) selection
For quick copying, press NVDA+Alt+V to enter copy mode, then press L to copy the current line or S to copy the entire screen.

Bookmarks

Set up to 10 bookmarks (0-9) in your terminal buffer for quick navigation.

Example Usage: When working with a long log file, you might bookmark important sections:
  1. Navigate to an error message at row 150
  2. Press NVDA+Alt+Shift+1 to set bookmark 1
  3. Navigate to another error at row 450
  4. Press NVDA+Alt+Shift+2 to set bookmark 2
  5. Later, press NVDA+Alt+1 to jump back to the first error instantly
  6. Press NVDA+Alt+2 to jump to the second error
Press NVDA+Alt+Shift+B to hear a list of all set bookmarks.

Command History Navigation

Scan and navigate through commands in your terminal session.

Example Usage: After running several commands, press NVDA+Alt+Shift+H to scan the terminal output and detect all commands (it looks for command prompts). Then use NVDA+Alt+Up and NVDA+Alt+Down to jump between commands. For example, if you ran ls, cd Documents, and dir, you can jump directly to each command and review its output. Press NVDA+Alt+Shift+L to hear a summary of all detected commands.

Output Search

Search terminal output for specific text patterns.

Example Usage: After running a test suite that produces lots of output, press NVDA+Alt+F to open the search dialog. Type "FAILED" to find all failed tests. Terminal Access will announce "Found 3 matches. Match 1 of 3: Test authentication FAILED". Press NVDA+F3 to jump to the next match and NVDA+Shift+F3 to go to the previous match. This is much faster than manually scanning through hundreds of lines.

Window Management

Define and monitor specific regions of the terminal screen.

Example Usage: In applications like htop or vim that have multiple screen regions, you can define a window to focus on specific areas:
  1. Navigate to the top-left corner of the region you want to monitor (e.g., the process list in htop)
  2. Press NVDA+Alt+F2 (you'll hear "Window start set")
  3. Navigate to the bottom-right corner of the region
  4. Press NVDA+Alt+F2 again (you'll hear "Window defined")
  5. Now press NVDA+Alt+Plus anytime to read just that region
Press NVDA+Alt+F3 to clear the window definition when done.

Punctuation Levels

Control how much punctuation and symbols are announced (None/Some/Most/All).

Example Usage: When reading prose or documentation in the terminal, press NVDA+Alt+[ to decrease to "Some" punctuation (only essential punctuation like periods and commas are announced). When working with code or configuration files, press NVDA+Alt+] repeatedly to increase to "Most" or "All" to hear every bracket, semicolon, and operator. For example, with "All" level, if (x > 5) { return; } announces every parenthesis, bracket, and semicolon.

Directional Reading

Read from cursor to any edge of the screen or buffer.

Example Usage: When you're in the middle of a long line and want to hear what comes next, press NVDA+Alt+Shift+Right to read from the cursor to the end of the line. If you're reviewing a long log file and want to hear everything from your current position to the end, press NVDA+Alt+Shift+Down to read to the bottom. Use NVDA+Alt+Shift+Left to read back to the beginning of the line and NVDA+Alt+Shift+Up to read to the top of the buffer.

Configurable Settings

Customize behavior through NVDA's settings dialog to match your workflow.

Example Usage: Access Terminal Access settings through NVDA menu > Preferences > Settings > Terminal Settings. You can adjust cursor tracking delay (useful if holding arrow keys causes too many announcements), enable symbol condensing (so "====" is announced as "4 equals" instead of repeating), customize which symbols to process, and more. See the Settings section below for detailed explanations.

Supported Terminals

Terminal Access for NVDA supports the following terminal applications on Windows 10 and Windows 11:

  • Windows Terminal
  • Windows PowerShell
  • PowerShell Core (pwsh)
  • Command Prompt (cmd.exe)
  • Console Host (conhost.exe)
Note: The add-on is designed to work with NVDA versions 2019.3 and later, ensuring compatibility with a wide range of NVDA installations.

Getting Started

Installation

  1. Download the Terminal Access for NVDA add-on file (.nvda-addon)
  2. Press Enter on the downloaded file to install it
  3. NVDA will ask for confirmation - press Enter to confirm
  4. Restart NVDA when prompted

First Use

When you open a supported terminal application for the first time after installing the add-on, NVDA will announce: "Terminal Access support active. Press NVDA+shift+f1 for help."

This announcement serves as a reminder that the add-on is active and provides a quick way to access this help documentation.

Practical Usage Scenarios

Here are real-world scenarios demonstrating how to use Terminal Access effectively:

Scenario 1: Reviewing Long Command Output

Situation: You ran pip list which displays 50+ installed packages.

Workflow:
  1. Press NVDA+Alt+PageUp to jump to the top of the output
  2. Use NVDA+Alt+A to have NVDA read all packages continuously
  3. If you hear a package name you want to examine, press any key to stop
  4. Use NVDA+Alt+I to re-read the current line
  5. Use NVDA+Alt+O and NVDA+Alt+U to review surrounding lines

Scenario 2: Debugging a Failed Command

Situation: Your build command failed with an error somewhere in hundreds of lines of output.

Workflow:
  1. Press NVDA+Alt+F to open search
  2. Type "error" or "failed" and press Enter
  3. Terminal Access announces: "Found 2 matches. Match 1 of 2: Build error in module auth"
  4. Press NVDA+F3 to jump to the next error
  5. Use NVDA+Alt+U and NVDA+Alt+O to read context around each error
  6. Press NVDA+Alt+Shift+1 to bookmark the first error location
  7. Navigate to the second error and bookmark it with NVDA+Alt+Shift+2
  8. Jump between bookmarked errors with NVDA+Alt+1 and NVDA+Alt+2

Scenario 3: Copying Specific Output

Situation: You need to copy an error message to share with your team.

Workflow:
  1. Navigate to the beginning of the error message
  2. Press NVDA+Alt+R to set the start mark
  3. Navigate line by line with NVDA+Alt+O until you reach the end of the error
  4. Press NVDA+Alt+R again to set the end mark
  5. Press NVDA+Alt+C to copy the selection
  6. Paste into your email or chat with Ctrl+V
Alternative: For copying a single line, press NVDA+Alt+V for copy mode, then press L to quickly copy the current line.

Scenario 4: Editing a Complex Command

Situation: You need to modify a long command with many parameters.

Workflow:
  1. Type or recall your command: docker run -d -p 8080:80 -v /data:/app/data --name myapp nginx
  2. Press Home to go to the beginning of the line
  3. Use NVDA+Alt+L repeatedly to move word by word: "docker", "run", "-d", "-p", etc.
  4. When you reach the word you want to change (e.g., "8080"), press NVDA+Alt+K to confirm
  5. Select the word with Ctrl+Shift+Right Arrow and type the new value
  6. Use NVDA+Alt+I to review the entire modified command before pressing Enter

Scenario 5: Working with Git Output

Situation: You ran git status and need to understand which files are modified.

Workflow:
  1. After running git status, press NVDA+Alt+Shift+H to scan command history
  2. The output shows "modified: src/app.py", "modified: tests/test_app.py", "untracked: README.md"
  3. Use NVDA+Alt+O to navigate line by line through the modified files
  4. For files of interest, use NVDA+Alt+R twice to select the filename
  5. Press NVDA+Alt+C to copy the filename to use in other commands
  6. Enable quiet mode with NVDA+Alt+Q before running git diff (produces lots of output)
  7. After the diff completes, disable quiet mode and use search to find specific changes

Scenario 6: Monitoring a Long-Running Process

Situation: You're running npm install which installs many packages with progress updates.

Workflow:
  1. Before running the command, press NVDA+Alt+Q to enable quiet mode
  2. Run npm install - the command executes without constant announcements
  3. Wait for the command to complete (you'll hear the new command prompt)
  4. Press NVDA+Alt+Q to disable quiet mode
  5. Press NVDA+Alt+PageDown to jump to the bottom (most recent output)
  6. Use NVDA+Alt+U repeatedly to review the last few lines and check for errors
  7. Or press NVDA+Alt+F and search for "error" or "warn" to find any issues

Scenario 7: Reviewing Python Code Indentation

Situation: You're viewing Python code in the terminal and need to understand its structure.

Workflow:
  1. Press NVDA+Alt+F5 to enable automatic indentation announcement
  2. Navigate to a function definition with NVDA+Alt+I
  3. Hear the line content followed by "No indentation" (top level function)
  4. Move to the next line with NVDA+Alt+O
  5. Hear the line content followed by "4 spaces" (function body)
  6. Continue navigating with indentation announced automatically after each line
  7. Or press NVDA+Alt+I twice to query indentation of current line only
  8. Use NVDA+Alt+Shift+0 through NVDA+Alt+Shift+9 to bookmark important functions
  9. Toggle off automatic announcement with NVDA+Alt+F5 when not needed

Scenario 8: Using Table Output (like htop or top)

Situation: You're using htop to monitor system processes.

Workflow:
  1. Launch htop
  2. Navigate to the top of the process list (skip header rows)
  3. Press NVDA+Alt+F2 to mark the window start
  4. Navigate to the bottom-right of the process list
  5. Press NVDA+Alt+F2 again to complete the window definition
  6. Now press NVDA+Alt+Plus anytime to hear just the process list
  7. Use NVDA+Alt+] to increase punctuation level if you need to hear percentage signs and brackets
  8. When done, press NVDA+Alt+F3 to clear the window

Scenario 9: Copying Columns from Tabular Data

Situation: You have a command output like ls -l with columns (permissions, size, date, filename) and only want the filenames.

Workflow:
  1. Navigate to the first filename column in the first row
  2. Press NVDA+Alt+P to note the column number (e.g., "Row 1, column 50")
  3. Press NVDA+Alt+R to set the start mark
  4. Navigate down to the last file with NVDA+Alt+O repeatedly
  5. Press Right Arrow to ensure you're at the same column position
  6. Press NVDA+Alt+R again to set the end mark
  7. Press NVDA+Alt+Shift+C for rectangular (column-based) copy
  8. You'll hear "Rectangular selection copied: 15 rows, columns 50 to 80"
  9. Now paste - you'll have just the filenames without other columns!

Scenario 10: Verifying Special Characters in Commands

Situation: You're typing a complex regex pattern and need to verify every special character.

Workflow:
  1. Open settings through NVDA menu > Preferences > Settings > Terminal Settings
  2. Enable "Process Symbols" so symbols are announced by name
  3. Set punctuation level to "All" for maximum detail
  4. Start typing your regex: grep -E '^[0-9]{3}-[0-9]{2}-[0-9]{4}$' file.txt
  5. As you type, hear: "g r e p space dash cap e space quote caret left-bracket 0 dash 9 right-bracket left-brace 3 right-brace..."
  6. After typing, press Home then use NVDA+Alt+Period repeatedly to review character by character
  7. For any character you're unsure about, press NVDA+Alt+Comma three times to hear its character code

Settings

Terminal Access settings can be accessed through NVDA's preferences dialog. Press NVDA+N to open the NVDA menu, select Preferences, then Settings, and choose the Terminal Settings category.

Available Settings

Enable Cursor Tracking

What it does: Automatically announces the character at the cursor position whenever the cursor moves in the terminal.

How it affects your experience: When enabled, you'll hear immediate feedback as you navigate through terminal text using arrow keys. This is particularly useful when:

  • Reviewing command output and wanting to track your position
  • Monitoring real-time updates in terminal applications
  • Navigating through text editors like vim or nano within the terminal
  • Following along as text appears during command execution

Behavior: Works with the "Cursor Delay" setting to control announcement timing. Respects "Quiet Mode" - no announcements when quiet mode is active. Uses "Process Symbols" setting to determine how symbols are announced.

When to disable: If you find cursor announcements distracting or if you're using navigation commands frequently and don't need position updates.

Default: Enabled

Enable Key Echo

What it does: Announces each character as you type it in the terminal.

How it affects your experience: Provides immediate auditory feedback for every keystroke, helping you:

  • Verify you've typed the correct character
  • Catch typos immediately as you type
  • Maintain confidence when entering commands or passwords (though passwords are typically not echoed by the terminal)
  • Work more efficiently without needing to review what you've typed

Behavior: Works together with "Process Symbols" - when both are enabled, symbols are announced by name (e.g., "dollar" instead of "$"). Works with "Condense Repeated Symbols" to avoid repetitive announcements when typing sequences like "====" or "----". Respects "Quiet Mode".

When to disable: If you're a confident typist and find character announcements slow down your workflow, or when typing large amounts of text where constant feedback becomes overwhelming.

Default: Enabled

Pause at Newlines

What it does: This setting is designed for NVDA's continuous reading functionality (speak all) to pause briefly at line breaks.

Current status: This setting is preserved for potential future use. Currently, the add-on focuses on manual navigation commands rather than continuous reading of terminal output.

Note: Line pauses during continuous reading are typically controlled by NVDA's core speech settings and synthesizer configuration.

Default: Enabled

Process Symbols

What it does: Speaks symbol characters by their name instead of the symbol itself.

How it affects your experience: When enabled, symbols are announced clearly:

  • "$" is announced as "dollar"
  • "@" is announced as "at"
  • "#" is announced as "hash"
  • "=" is announced as "equals"
  • And many more (see full list of 25+ symbols in the code)

Affects: Both typing (with Key Echo) and navigation (when reading characters with NVDA+Alt+Comma/Period). Also affects cursor tracking announcements.

When to enable:

  • Working with shell scripts that use many special characters
  • Editing configuration files with complex syntax
  • Writing code or reviewing command-line parameters
  • When you need to distinguish between similar-sounding symbols

When to disable: When symbols are obvious from context or when you prefer hearing the actual symbol pronunciation for faster comprehension.

Default: Disabled

Condense Repeated Symbols

What it does: Counts consecutive identical symbols and announces them as a group (e.g., "3 equals" instead of "equals equals equals").

How it affects your experience: Significantly improves the experience when typing:

  • Horizontal rules in comments: ---- becomes "4 dash" instead of "dash dash dash dash"
  • Separators: ==== becomes "4 equals"
  • Markdown headers: ### becomes "3 hash"
  • Underscores in variable names: __init__ announces underscores as "2 underscore"

Behavior: Only condenses symbols specified in "Repeated Symbols to Condense" setting. Works during typing (Key Echo must be enabled). Uses "Process Symbols" to determine how to name the symbols.

Technical note: Announcements are deferred until a different character is typed, so you won't hear the count until you type something else. Single instances are announced normally.

When to enable: When frequently typing repetitive symbols like creating ASCII art, writing formatted text, or working with code that uses separator lines.

Default: Disabled

Repeated Symbols to Condense

What it does: Specifies exactly which symbols should be condensed when "Condense Repeated Symbols" is enabled.

How to configure: Enter symbols as a continuous string without spaces. Each character in the string will be tracked for repetition.

Examples:

  • -_=! (default) - Condenses dashes, underscores, equals, and exclamation marks
  • -=# - For Markdown users who frequently type headers and separators
  • *-_ - For emphasis markers in text
  • -_=!@#$ - Extended set for programming

Behavior: Only active when "Condense Repeated Symbols" is enabled. Does not affect characters not in this list - those are announced individually as normal.

Default: -_=!

Cursor Delay (milliseconds)

What it does: Sets how long to wait after cursor movement before announcing the character at the new position.

How it affects your experience: This delay acts as a "debounce" to prevent overwhelming speech during rapid cursor movement:

  • Low delay (0-50ms): Nearly instant announcements. Best if you move the cursor deliberately and want immediate feedback. May result in rapid-fire announcements when holding arrow keys.
  • Medium delay (50-100ms): Balanced responsiveness. Good for most users. Filters out very rapid movements while still feeling responsive.
  • High delay (100-1000ms): Waits longer before announcing. Useful when you frequently hold arrow keys to move across long lines and only want to hear the final position.

Technical behavior: Each new cursor movement resets the timer. NVDA only announces when the cursor stays in one position for the full delay duration.

Recommended values:

  • Arrow key navigation: 20-50ms (default: 20ms)
  • Frequent long-distance moves: 100-200ms
  • Following rapid updates: 50-100ms

Note: Only affects cursor tracking. Does not affect key echo or manual navigation commands.

Default: 20 milliseconds

Range: 0 to 1000 milliseconds

Settings Interaction Summary:
  • Quiet Mode (toggled with NVDA+Alt+Q) temporarily disables cursor tracking and key echo, regardless of their settings
  • Process Symbols affects cursor tracking, key echo, and manual character navigation
  • Condense Repeated Symbols only works with Key Echo enabled
  • Cursor Delay only affects cursor tracking announcements

Advanced Topics

Application Profiles

Application profiles allow Terminal Access to automatically adjust its settings based on the terminal application you're using. Each profile can customize punctuation levels, cursor tracking modes, and define window regions for specialized behavior.

Built-in Application Profiles

Terminal Access comes with default profiles for popular applications:

  • vim/nvim: Optimized for Vim/Neovim editors
    • Punctuation: MOST (for code symbols)
    • Cursor Tracking: WINDOW mode
    • Silent zones: Status line (bottom line), Command line (second from bottom)
  • tmux: Terminal multiplexer support
    • Cursor Tracking: STANDARD mode
    • Silent zones: Status bar (bottom line)
  • htop: Process viewer optimization
    • Repeated symbols: Disabled (progress bars have many repeated characters)
    • Window regions: Header (lines 1-4), Process list (lines 5+)
  • less/more: Pager applications
    • Quiet mode: Enabled
    • Key echo: Disabled (navigation keys not announced)
  • git: Version control operations
    • Punctuation: MOST (symbols in diffs)
    • Repeated symbols: Disabled (many dashes/equals)
  • nano: GNU nano editor
    • Cursor Tracking: STANDARD mode
    • Silent zones: Status bar, Shortcut bar (bottom two lines)
  • irssi: IRC client
    • Punctuation: SOME (for chat)
    • Line pause: Disabled (fast reading for chat)
    • Silent zones: Status bar (bottom line)
  • WSL: Windows Subsystem for Linux
    • Punctuation: MOST (code-friendly for Linux commands and paths)
    • Cursor Tracking: STANDARD mode
    • Repeated symbols: OFF (progress bars and separators common in Linux tools)

Managing Profiles

To view and manage installed profiles:

  1. Open NVDA Settings (NVDA+N → Preferences → Settings)
  2. Navigate to "Terminal Access" category
  3. Go to "Application Profiles" section
  4. The "Installed profiles" dropdown shows all available profiles

Exporting a Profile:

  1. Open Terminal Access Settings
  2. Navigate to "Application Profiles" section
  3. Select the profile you want to export from the dropdown
  4. Click "Export..." button
  5. Choose a location and filename (default: profilename_profile.json)
  6. Click "Save"

Importing a Profile:

  1. Open Terminal Access Settings
  2. Navigate to "Application Profiles" section
  3. Click "Import..." button
  4. Browse to the profile JSON file
  5. Click "Open"

Deleting a Custom Profile:

  1. Open Terminal Access Settings
  2. Navigate to "Application Profiles" section
  3. Select the custom profile from the dropdown
  4. Click "Delete Profile" button
  5. Confirm deletion
Note: Default profiles (vim, tmux, htop, less, git, nano, irssi, WSL) cannot be deleted.

Third-Party Terminal Support

Terminal Access supports 13 popular third-party terminal emulators in addition to the 5 built-in Windows terminals.

Supported Third-Party Terminals

  • Cmder: Portable console emulator for Windows
    • Popular among developers
    • Includes Unix tools
    • Default profile: Balanced settings for general use
  • ConEmu: Windows console emulator with tabs
    • Both 32-bit and 64-bit versions supported
    • Highly customizable
    • Supports multiple console processes
  • mintty: Terminal emulator for Git Bash and Cygwin
    • Lightweight and fast
    • Popular for Git operations
    • Default profile: MOST punctuation (for development)
  • PuTTY / KiTTY: SSH and telnet clients
    • Industry-standard for remote access
    • Optimized for SSH sessions

Window Definitions

Window definitions allow you to define specific regions of the terminal screen with different speech behaviors. This is useful for applications with status bars, command areas, or split panes.

Window Definition Basics

Each window definition has:

  • Name: Identifier for the window
  • Coordinates: Top, bottom, left, right (1-based)
  • Mode: How content is announced
  • Enabled: Whether the window is active

Window Modes

  • announce: Read content normally (default)
  • silent: Suppress all speech for this region
  • monitor: Track changes but announce differently

Coordinate System

Coordinates are 1-based (row 1, col 1 is top-left):

  • Top/Bottom: Row numbers (1 to screen height)
  • Left/Right: Column numbers (1 to screen width)
  • 9999: Special value meaning "last row/column"

Use Cases

  • Status Bars: Silence repetitive status information
  • Split Panes: Define regions for tmux/screen panes
  • Headers: Special handling for htop/top headers
  • Command Areas: Monitor command input regions

Unicode and International Text

CJK Character Support

Terminal Access correctly handles double-width characters used in Chinese, Japanese, and Korean:

  • Accurate Width Calculation: CJK characters count as 2 columns
  • Column Extraction: Rectangular selection works correctly with CJK
  • Combining Characters: Zero-width combining marks handled properly
Example: In the text "Hello世界", "Hello" = 5 columns, "世界" = 4 columns, total = 9 columns

Right-to-Left (RTL) Text Support

Terminal Access automatically detects and processes RTL text:

Supported Languages:

  • Arabic (U+0600-U+06FF, U+0750-U+077F)
  • Hebrew (U+0590-U+05FF)

Features:

  • Automatic Detection: Analyzes character ranges to detect RTL text
  • Bidirectional Algorithm: Unicode UAX #9 implementation
  • Arabic Reshaping: Contextual forms (initial, medial, final, isolated)
  • Mixed Text: Handles RTL and LTR text together
  • Column Extraction: RTL-aware column operations
Optional Dependencies: For full RTL support, install:
pip install python-bidi arabic-reshaper
Without these libraries, Terminal Access gracefully degrades to basic Unicode support.

Emoji Support

Terminal Access handles complex emoji sequences:

Supported Features:

  • Zero-Width Joiners (ZWJ): Family emoji, profession emoji
  • Skin Tone Modifiers: U+1F3FB through U+1F3FF
  • Variation Selectors: Emoji vs. text presentation
  • Flag Sequences: Country flags
  • Width Calculation: Emoji typically 2 columns wide
Optional Dependency: For full emoji support, install:
pip install emoji

Performance Optimization

Terminal Access includes several performance optimizations for large terminal buffers.

Position Caching

Position calculations are cached for fast repeated access:

  • Cache Timeout: 1000ms (1 second)
  • Cache Size: Up to 100 positions
  • Automatic Invalidation: On content changes, window resize, terminal switch

Performance Impact:

  • First calculation: O(n) where n = row number
  • Cached calculation: O(1) constant time
  • Row 1000: ~500ms → <1ms with cache

Incremental Position Tracking

For small cursor movements (within 10 positions):

  • 10-20x faster than full calculation
  • No cache required for simple movements
  • Automatic fallback for large jumps

Background Processing

Large rectangular selections (>100 rows) run in background threads:

  • Progress Dialog: Shows completion percentage
  • Cancellation Support: Cancel long-running operations
  • Operation Queue: Prevents overlapping operations

Tips and Best Practices

Efficient Terminal Usage

  • Use Quiet Mode: When running commands that produce verbose output, enable quiet mode (NVDA+Alt+Q) to prevent overwhelming speech output.
  • Review Output: After a command completes, use line navigation (NVDA+Alt+U/I/O) to review the output at your own pace.
  • Word Navigation: When examining long command lines or paths, word navigation (NVDA+Alt+J/K/L) is faster than character-by-character review.
  • Phonetic Spelling: When dealing with complex filenames or unclear output, use phonetic character reading (NVDA+Alt+Comma twice) to ensure accuracy.

Customization

  • Adjust Cursor Delay: If you find cursor tracking announcements too frequent or too slow, adjust the cursor delay setting to match your preference.
  • Symbol Processing: Enable symbol processing when working with scripts or configuration files to better understand punctuation and special characters.
  • Key Echo: If you're a confident typist, you may prefer to disable key echo to reduce speech output during typing.

Troubleshooting

Add-on Not Working

Symptom: Commands don't work or NVDA doesn't announce terminal support.

Solution:

  • Ensure you're using a supported terminal application
  • Restart NVDA (NVDA+Q, then restart)
  • Verify the add-on is installed (NVDA menu > Tools > Manage add-ons)
  • Check that the add-on is enabled in the add-ons manager

Help File Doesn't Open

Symptom: Pressing NVDA+Shift+F1 doesn't open the user guide.

Solution:

  • Ensure the add-on is properly installed
  • Try reinstalling the add-on
  • Check that your default browser or HTML viewer is properly configured

Commands Conflict with Other Add-ons

Symptom: Terminal Access commands trigger unexpected behavior.

Solution:

  • Check for keyboard shortcut conflicts in NVDA's Input Gestures dialog
  • Disable conflicting add-ons temporarily to identify the issue
  • Remap conflicting gestures through Input Gestures (NVDA menu > Preferences > Input Gestures)

Settings Not Saving

Symptom: Changes to Terminal Access settings don't persist after restart.

Solution:

  • Ensure NVDA has write permissions to its configuration directory
  • Check for NVDA running with elevated privileges issues
  • Try resetting NVDA configuration (backup first!)
